# tunnel-agent

Run AI coding agents inside a Docker container on your China server, routing all traffic through WireGuard to a VPS you control.

Supports **8 agents** out of the box: Claude Code, Codex, Aider, Goose, Cline, Gemini CLI, Amp, OpenCode.

---

## The Problem

Running both an SSL VPN (to connect to your China server) and a separate VPN for AI API access simultaneously causes routing conflicts. You can't reach Anthropic's API from inside the China server without a second tunnel, and you can't run two full-system VPNs at once.

## The Solution

tunnel-agent runs the agent inside a Docker container on the China server. The container starts a WireGuard tunnel to a VPS you control (e.g. a $5/mo Vultr node in Tokyo). All traffic from the container — including AI API calls — exits through the VPS. Your SSL VPN connection to the China server stays unaffected. Claude Code can also access server files directly via the mounted workspace.

---

## Architecture

```
Your Mac → SSL VPN → China Server
                       └─ Docker container
                            ├─ WireGuard → VPS (Tokyo) → Anthropic API
                            └─ Claude Code → server files directly
```

The Docker container runs with `NET_ADMIN` and `SYS_MODULE` capabilities so WireGuard can create the TUN device and manage routing inside the container. No changes are needed to the host network stack.

---

## Prerequisites

- **Docker** (on the China server) with Compose v2
- **Python 3.11+** (on the China server)
- **A VPS** with WireGuard configured — see VPS Setup below
- **A WireGuard `.conf` file** for the client peer (generated by the VPS setup script)
- **Auth** for your chosen agent — Claude Code subscription (via `~/.claude/`) or an API key

---

## VPS Setup

Any Linux VPS works. Vultr's $5/mo Tokyo instance is a common choice for low latency to Anthropic's API from China.

1. Provision a fresh Ubuntu 22.04 VPS
2. SSH in and run the WireGuard installer:

```bash
curl -O https://raw.githubusercontent.com/angristan/wireguard-install/master/wireguard-install.sh
chmod +x wireguard-install.sh
sudo ./wireguard-install.sh
```

3. Follow the prompts. When it asks for a client name, use something like `docker-client`.
4. The script generates a `.conf` file (e.g. `/root/wg0-client-docker-client.conf`).
5. Copy that file to the China server:

```bash
scp root@your-vps:/root/wg0-client-docker-client.conf ~/.tunnel-agent/wg0.conf
```

The default config path is `~/.tunnel-agent/wg0.conf`. You can override it with `--wg-config` or in the config file.

## Quick Start

```bash
# Run Claude Code with the default WireGuard config (~/.tunnel-agent/wg0.conf)
tunnel-agent run --agent claude

# Pass a task directly
tunnel-agent run --agent claude "fix the auth bug in server.py"

# Use a non-default WireGuard config file
tunnel-agent run --agent claude --wg-config /path/to/custom.conf
```

Claude Code mounts `~/.claude/` from the host automatically, so your subscription tokens carry over with no extra setup.

## CLI Reference

### `tunnel-agent run`

```
tunnel-agent run [OPTIONS] [TASK]
```

Launch an agent in a WireGuard tunnel container. If `TASK` is omitted, starts in interactive mode.

| Flag | Default | Description |
|------|---------|-------------|
| `--agent NAME` | `claude` | Agent to run. See `tunnel-agent agents` for valid names. |
| `--workspace PATH` | `.` (current dir) | Directory to mount as `/workspace` in the container. |
| `--wg-config PATH` | `~/.tunnel-agent/wg0.conf` | Path to the WireGuard client `.conf` file. |
| `--verbose` | — | Enable debug logging. |

### `tunnel-agent agents`

```
tunnel-agent agents
```

List all supported agents with their CLI name and install method.

### `tunnel-agent config`

```
tunnel-agent config [OPTIONS]
```

View or update the config file at `~/.tunnel-agent/config.yaml`.

| Flag | Description |
|------|-------------|
| `--wg-config PATH` | Set the default WireGuard config path |
| `--default-agent NAME` | Set the default agent |
| *(no flags)* | Print the current config |

## Configuration

tunnel-agent reads `~/.tunnel-agent/config.yaml` on startup. If the file does not exist, defaults are used. You can create and edit it by hand or use `tunnel-agent config`.

```yaml
wireguard:
  config_path: ~/.tunnel-agent/wg0.conf   # Path to the WireGuard client .conf file

default_agent: claude          # Used when --agent is not specified
mount_ssh: true                # Mount ~/.ssh into the container
mount_claude: true             # Mount ~/.claude into the container
```

All fields are optional — missing fields fall back to defaults.

## Python API

```python
from tunnel_agent import TunnelBackend, TunnelConfig, WireGuardConfig
from isolated_agent import ClaudeCodeAgent, Session

# Use defaults (WireGuard config at ~/.tunnel-agent/wg0.conf)
backend = TunnelBackend(config=TunnelConfig())

# Or specify a custom config path
backend = TunnelBackend(config=TunnelConfig(
    wireguard=WireGuardConfig(config_path="/path/to/custom.conf"),
    mount_ssh=True,
    mount_claude=True,
))

session = Session(
    agent=ClaudeCodeAgent(),
    backend=backend,
    workspace="./my-project",
)
result = session.run(task="fix the auth bug")
print(f"Exit code: {result.exit_code}, Duration: {result.duration_seconds}s")
```

## How It Works

### Container startup sequence

When `tunnel-agent run` is invoked, the backend:

1. Renders a Dockerfile, `docker-compose.yml`, `entrypoint.sh`, and the WireGuard config into a temp build directory
2. Builds the Docker image (Debian base with `wireguard-tools`, `iproute2`, `su-exec`, `openssh-client`, and the agent CLI)
3. Starts the container and waits for it to be healthy

Inside the container, `entrypoint.sh` runs as root and performs these steps before handing off to the agent:

1. **Create TUN device** — creates `/dev/net/tun` if not present (required for WireGuard)
2. **Start WireGuard** — runs `wg-quick up wg0`, which configures the tunnel interface and installs routing rules for all traffic
3. **Fix SSH key permissions** — the `~/.ssh` directory is mounted read-only at `.ssh-mount` to avoid host permission conflicts, then copied with `chmod 600` applied
4. **Source environment variables** — loads `/app/.env` into the environment
5. **Drop to non-root and launch agent** — `exec gosu <user> <agent-cli> [task]`

## Limitations

**Requires a VPS.** You need a server outside China with WireGuard installed. The $5/mo Vultr Tokyo plan covers typical AI coding workloads.

**All container traffic goes through the VPS.** Unlike selective routing, every packet from the container exits via WireGuard. This includes git pushes, package downloads, and anything else the agent does.

**The `.conf` file must be set up manually.** Run the WireGuard install script on your VPS, copy the generated client config to `~/.tunnel-agent/wg0.conf`, and you're done. There is no automated key exchange.

**Claude subscription auth depends on `~/.claude/` contents.** If your OAuth tokens are stored only in macOS Keychain (not written to `~/.claude/`), you will need to run `claude login` inside the container on first use.
